Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clients are the typical Web user's Internet-connected devices (like your computer or phone) and the web-accessing software on those devices (usually a browser like Firefox or Chrome).
  • Servers are computers that store webpages, sites, or apps. When a client wants to access a webpage, a copy is downloaded from the server to the client and displayed in the browser.

So What Actually Happens?

  1. The browser goes to the DNS server to find the real address of the server hosting the website.
  2. The browser sends an HTTP request to the server asking for a copy of the website.
  3. If approved, the server sends a 200 OK message and begins sending the website’s files in chunks.
  4. The browser assembles the chunks into a complete website and displays it to you!

The Big 3 – HTML, CSS, JavaScript

While the server can process information in many different languages, the files that they serve to the client are always going to be some combination of HTML, CSS, and JavaScript!
